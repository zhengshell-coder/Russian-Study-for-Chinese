# 多邻国式学习引擎适配方案 v1

## 1. 目标

在保留当前个人自用、轻量、弱游戏化方向的前提下，把现有俄语学习工具从“每日内容展示 + 复习确认”升级为“线性路径 + 小课题型引擎 + 即时反馈”的学习系统。

本方案不引入社交、排行榜、虚拟货币、复杂动画地图，也不恢复手写需求。

## 2. 当前实现判断

当前系统已经具备：
- 全年 `24` 个周期、每周期 `12` 天的长期课程框架
- 每天 `part1 基础 / part2 场景 / part3 跟读 / 复习检查`
- 俄语、发音标注、中文释义分字段保存
- 音频播放、慢速播放、基础跟读
- 本地进度保存、PWA、GitHub Pages 发布

当前主要缺口：
- 还没有真正的题型引擎
- 学习页仍偏“内容卡片”，不是“做题卡片”
- 复习检查仍偏主观确认，不是即时反馈题
- 课程数据还没有模板化语法生成能力
- 每课没有明确的 `8-12` 题进度条和完成反馈

## 3. 适配原则

### 3.1 保留
- 单一路径学习
- 小步推进
- 手机单卡片界面
- 音频慢速 / 正常播放
- 中文语义锚点
- 本地保存学习进度

### 3.2 增强
- 题型系统
- 即时反馈
- 每课进度条
- 错题回收
- 模板化句型生成

### 3.3 暂不做
- 生命值
- 宝箱奖励
- 排行榜
- 社交功能
- 复杂地图动画
- 虚拟货币

## 4. 新目标架构

```text
Course
  └── Cycle
        └── Day
              └── Lesson
                    └── Exercise[]
```

当前 `Day` 仍然保留，但 `part1 / part2 / part3 / review` 会逐步转为同一个 `Exercise Engine` 渲染。

## 5. 题型优先级

### 5.1 MVP 第一批必须做

#### 1. 单词匹配题

用途：
- 建立中文与俄语词汇映射
- 适合 `part1 基础`

数据结构：

```js
{
  type: "match_pairs",
  prompt: "匹配俄语和中文",
  pairs: [
    { ru: "мама", ipa: "[MA-ma]", zh: "妈妈" }
  ]
}
```

#### 2. 句子拼装题

用途：
- 训练语序和句型
- 适合 `part2 场景`

数据结构：

```js
{
  type: "sentence_builder",
  prompt: "拼出这句话",
  answer: "Я хочу чай.",
  ipa: "[ya ha-CHOO chai]",
  zh: "我想要茶。",
  tokens: ["чай", "Я", "хочу"]
}
```

#### 3. 听音排序题

用途：
- 训练听辨和词序
- 比完整听写更适合初学者

数据结构：

```js
{
  type: "listen_order",
  prompt: "按听到的顺序点词",
  answer: "Я хочу чай.",
  ipa: "[ya ha-CHOO chai]",
  zh: "我想要茶。",
  tokens: ["чай", "хочу", "Я"]
}
```

#### 4. 即时反馈卡

所有题型统一反馈：

```text
正确 / 再试一次
正确答案
Meaning: 中文解释
继续
```

### 5.2 第二批实现

#### 5. 听写输入题

用途：
- 训练拼写和主动回忆

先做宽松匹配：
- 忽略大小写
- 忽略句号
- 允许 `ё` 和 `е` 作为可配置等价

#### 6. 跟读题升级

用途：
- 保留当前 `part3 跟读`
- 增加“跳过说话 / 当前不方便说话”

### 5.3 后置

- 生命值
- 连击
- 宝箱
- 复杂能力评估

## 6. 每日结构改造

### 6.1 当前结构

```text
part1 基础：看核心词
part2 场景：看场景句
part3 跟读：听和读
复习检查：确认是否记住
```

### 6.2 目标结构

```text
Lesson：8-12 个 Exercise
  1-2 题：词汇识别
  3-5 题：匹配 / 选择
  6-8 题：句子拼装
  9-10 题：听音排序
  11-12 题：跟读或复习
```

保留页面上的 `part1 / part2 / part3`，但它们的定位改为：
- `part1 基础`：词汇识别与匹配
- `part2 场景`：句子拼装与听音排序
- `part3 跟读`：单词跟读与整句跟读

## 7. 渐进式难度规则

每个句型模板按以下顺序训练：

```text
recognition → assisted recall → production → speech
```

对应题型：

```text
单词匹配 → 句子拼装 → 听写输入 → 跟读
```

## 8. 语法模板引擎规划

### 8.1 初始模板

```js
{
  id: "want_object",
  pattern: "SUBJECT + VERB_WANT + OBJECT",
  meaning: "我想要……",
  slots: {
    SUBJECT: ["Я"],
    VERB_WANT: ["хочу"],
    OBJECT: ["чай", "кофе", "воду"]
  }
}
```

### 8.2 否定模板

```js
{
  id: "not_want_object",
  pattern: "SUBJECT + не + VERB_WANT + OBJECT",
  meaning: "我不想要……",
  slots: {
    SUBJECT: ["Я"],
    VERB_WANT: ["хочу"],
    OBJECT: ["чай", "кофе", "молоко"]
  }
}
```

### 8.3 第一阶段只做半自动

先不做完全自动生成。课程数据中显式保存句子，同时附加 `templateId`。这样更安全，方便人工校对俄语。

## 9. 手机界面适配方向

### 9.1 页面模型

从“part 内卡片列表”转为：

```text
顶部：第 X 轮 · 第 Y 天
进度条：当前 Lesson 完成度
中间：单题卡片
底部：检查 / 继续
```

### 9.2 操作原则

- 一屏只做一件事
- 不让多个主按钮挤在同一区域
- `听` 固定靠近题干
- `检查 / 继续` 固定在底部
- 错误反馈覆盖在当前题下方，不跳页
- 所有选项使用大触控块

## 10. 数据结构迁移方案

### 10.1 新增字段

在每日课程中新增：

```js
exercises: [
  {
    id: "day-1-ex-1",
    part: "foundation",
    type: "match_pairs",
    prompt: "匹配俄语和中文",
    payload: {}
  }
]
```

### 10.2 兼容旧字段

短期保留：
- `foundation`
- `scene`
- `voice`
- `review`

新引擎优先读取 `exercises`。如果某天没有 `exercises`，继续使用旧渲染。

## 11. 实施顺序

### 阶段 1：Exercise Engine 骨架

目标：
- 支持单题渲染
- 支持进度条
- 支持检查与继续
- 支持即时反馈

范围：
- 先接入 `match_pairs`
- 先改第 `1` 天作为样板

### 阶段 2：句子拼装题

目标：
- 支持 token bank
- 支持答案槽
- 支持正确 / 错误反馈

范围：
- 接入第 `1-3` 天

### 阶段 3：听音排序题

目标：
- 音频播放
- 慢速播放
- 点词排序

范围：
- 接入第 `4-6` 天

### 阶段 4：复习检查题型化

目标：
- 复习不再只是“是否记住”
- 错题进入本地错题队列

范围：
- 替换普通日复习
- 第 `12` 天保留整轮复盘逻辑，但内部改题型

### 阶段 5：模板化课程扩展

目标：
- 让 `Я хочу X`、`Я не хочу X` 等模板驱动多题生成
- 建立 A1 到 A2 的课程语法路线

## 12. 对当前项目的结论

当前项目不应继续在“卡片展示 UI”上反复微调。下一步最有价值的升级是建立 `Exercise Engine`。

建议先做最小闭环：

```text
第 1 天
  题 1：单词匹配
  题 2：听音选择
  题 3：句子拼装
  题 4：听音排序
  题 5：跟读
  题 6：复习确认
```

只要这个闭环跑通，后续全年课程就可以逐步从“内容数据”迁移成“题型数据”。
