# 多邻国式学习引擎适配方案 v1

## 1. 目标

在保留当前个人自用、轻量、弱游戏化方向的前提下，把现有俄语学习工具从“每日内容展示 + 复习确认”升级为“线性路径 + 小课题型引擎 + 即时反馈”的学习系统。

本方案不引入社交、排行榜、虚拟货币、复杂动画地图，也不恢复手写需求。

## 2. 当前实现判断

当前系统已经具备：
- 全年 `24` 个周期、每周期 `12` 天的长期课程框架
- 每天 `part1 基础 / part2 场景 / part3 跟读 / 复习检查`
- 俄语、发音标注、中文释义分字段保存
- 音频播放、慢速播放、基础跟读
- 本地进度保存、PWA、GitHub Pages 发布

当前主要缺口：
- 还没有真正的题型引擎
- 学习页仍偏“内容卡片”，不是“做题卡片”
- 复习检查仍偏主观确认，不是即时反馈题
- 课程数据还没有模板化语法生成能力
- 每课没有明确的 `8-12` 题进度条和完成反馈

## 2.1 原开发逻辑不可破坏项

后续升级必须保持以下一致性：

- 全年仍是 `24` 个周期，每周期 `12` 天
- 每周期第 `12` 天仍是整轮复盘
- 每天外层仍保留 `part1 基础 / part2 场景 / part3 跟读 / 复习检查`
- 底部 `上一天 / 下一天` 只切换整天，不切换题目
- 手机端保持单屏优先，不恢复长列表课程页
- 手写需求继续排除
- 俄语字段继续遵守标准西里尔字母规则
- 所有俄语词句继续保留发音标注和中文语义锚点

也就是说，新题型引擎不是推翻当前结构，而是替换每个 `part` 内部的学习方式。

## 3. 适配原则

### 3.1 保留
- 单一路径学习
- 小步推进
- 手机单卡片界面
- 音频慢速 / 正常播放
- 中文语义锚点
- 本地保存学习进度

### 3.2 增强
- 题型系统
- 即时反馈
- 每课进度条
- 错题回收
- 模板化句型生成

### 3.3 暂不做
- 生命值
- 宝箱奖励
- 排行榜
- 社交功能
- 复杂地图动画
- 虚拟货币

## 4. 新目标架构

```text
Course
  └── Cycle
        └── Day
              └── Lesson
                    └── Exercise[]
```

当前 `Day` 仍然保留，但 `part1 / part2 / part3 / review` 会逐步转为同一个 `Exercise Engine` 渲染。

## 4.1 兼容迁移架构

实际开发采用双轨兼容：

```text
Day
  ├── legacy content
  │     ├── foundation
  │     ├── scene
  │     ├── voice
  │     └── review
  └── exercises
        ├── foundation[]
        ├── scene[]
        ├── voice[]
        └── review[]
```

渲染规则：

```text
如果当前 part 有 exercises → 使用 Exercise Engine
如果当前 part 没有 exercises → 使用旧卡片渲染
```

这样可以先改第 `1` 天，不影响全年其他 `287` 天。

## 5. 题型优先级

### 5.1 MVP 第一批必须做

#### 1. 单词匹配题

用途：
- 建立中文与俄语词汇映射
- 适合 `part1 基础`

数据结构：

```js
{
  type: "match_pairs",
  prompt: "匹配俄语和中文",
  pairs: [
    { ru: "мама", ipa: "[MA-ma]", zh: "妈妈" }
  ]
}
```

#### 2. 句子拼装题

用途：
- 训练语序和句型
- 适合 `part2 场景`

数据结构：

```js
{
  type: "sentence_builder",
  prompt: "拼出这句话",
  answer: "Я хочу чай.",
  ipa: "[ya ha-CHOO chai]",
  zh: "我想要茶。",
  tokens: ["чай", "Я", "хочу"]
}
```

#### 3. 听音排序题

用途：
- 训练听辨和词序
- 比完整听写更适合初学者

数据结构：

```js
{
  type: "listen_order",
  prompt: "按听到的顺序点词",
  answer: "Я хочу чай.",
  ipa: "[ya ha-CHOO chai]",
  zh: "我想要茶。",
  tokens: ["чай", "хочу", "Я"]
}
```

#### 4. 即时反馈卡

所有题型统一反馈：

```text
正确 / 再试一次
正确答案
Meaning: 中文解释
继续
```

### 5.2 第二批实现

#### 5. 听写输入题

用途：
- 训练拼写和主动回忆

先做宽松匹配：
- 忽略大小写
- 忽略句号
- 允许 `ё` 和 `е` 作为可配置等价

#### 6. 跟读题升级

用途：
- 保留当前 `part3 跟读`
- 增加“跳过说话 / 当前不方便说话”

### 5.3 后置

- 生命值
- 连击
- 宝箱
- 复杂能力评估

## 6. 每日结构改造

### 6.1 当前结构

```text
part1 基础：看核心词
part2 场景：看场景句
part3 跟读：听和读
复习检查：确认是否记住
```

### 6.2 目标结构

```text
Lesson：8-12 个 Exercise
  1-2 题：词汇识别
  3-5 题：匹配 / 选择
  6-8 题：句子拼装
  9-10 题：听音排序
  11-12 题：跟读或复习
```

保留页面上的 `part1 / part2 / part3`，但它们的定位改为：
- `part1 基础`：词汇识别与匹配
- `part2 场景`：句子拼装与听音排序
- `part3 跟读`：单词跟读与整句跟读

## 6.3 不采用的改法

不直接把每天改成一个无分区的 `8-12` 题长 lesson。

原因：
- 用户已经熟悉当前 `part1 / part2 / part3` 结构
- 手机上无分区题流容易失去“我在学什么”的方向感
- 第 `12` 天复盘需要继续按天分组
- 当前全年数据已经围绕 `foundation / scene / voice / review` 建立

正确做法：
- 外层保持 `part`
- 内层逐步题型化
- 每个 `part` 内最多 `2-4` 个小题，避免过长

## 6.4 单日目标题量

在每日 `20-30` 分钟范围内，建议：

```text
part1 基础：2-3 题
part2 场景：3-4 题
part3 跟读：2-3 题
复习检查：2-3 题
```

总题量控制在 `8-12` 题。

## 7. 渐进式难度规则

每个句型模板按以下顺序训练：

```text
recognition → assisted recall → production → speech
```

对应题型：

```text
单词匹配 → 句子拼装 → 听写输入 → 跟读
```

## 8. 语法模板引擎规划

### 8.1 初始模板

```js
{
  id: "want_object",
  pattern: "SUBJECT + VERB_WANT + OBJECT",
  meaning: "我想要……",
  slots: {
    SUBJECT: ["Я"],
    VERB_WANT: ["хочу"],
    OBJECT: ["чай", "кофе", "воду"]
  }
}
```

### 8.2 否定模板

```js
{
  id: "not_want_object",
  pattern: "SUBJECT + не + VERB_WANT + OBJECT",
  meaning: "我不想要……",
  slots: {
    SUBJECT: ["Я"],
    VERB_WANT: ["хочу"],
    OBJECT: ["чай", "кофе", "молоко"]
  }
}
```

### 8.3 第一阶段只做半自动

先不做完全自动生成。课程数据中显式保存句子，同时附加 `templateId`。这样更安全，方便人工校对俄语。

## 8.4 场景化课程规则

多邻国式场景化不是“主题词表”，而是：

```text
场景目标 → 核心词块 → 最小句型 → 句型变体 → 听说复用
```

每个场景必须有一个可执行目标。

示例：

```text
场景：点饮料
目标：能说“我想要茶 / 咖啡 / 水”
part1：чай / кофе / вода
part2：Я хочу чай. / Я не хочу кофе.
part3：单词跟读 + 整句跟读
review：听音排序或句子拼装
```

不合格设计：

```text
part1 放一堆饮料词
part2 再放一堆无关句子
```

合格设计：

```text
part1 的词必须在 part2 中被使用
part2 的句子必须在 part3 中被跟读
复习题必须回收 part1 和 part2 的内容
```

## 8.5 语音方案规则

语音学习拆为三层：

### 1. 输入层

目标：先听懂。

题型：
- 慢速听
- 正常听
- 听音排序
- 听音选择

### 2. 模仿层

目标：能跟读。

题型：
- 单词跟读
- 整句跟读
- 慢速跟读

### 3. 输出层

目标：能主动说出。

题型：
- 看中文说俄语
- 听中文提示后说俄语

MVP 先做输入层和模仿层，不急着做专业纠音。

每个语音题必须提供：
- 正常播放
- 慢速播放
- 中文含义
- 跳过或“现在不方便说”

## 9. 手机界面适配方向

### 9.1 页面模型

从“part 内卡片列表”转为：

```text
顶部：第 X 轮 · 第 Y 天
进度条：当前 Lesson 完成度
中间：单题卡片
底部：检查 / 继续
```

### 9.2 操作原则

- 一屏只做一件事
- 不让多个主按钮挤在同一区域
- `听` 固定靠近题干
- `检查 / 继续` 固定在底部
- 错误反馈覆盖在当前题下方，不跳页
- 所有选项使用大触控块

### 9.3 与当前 UI 的一致性

当前顶部结构继续保留：

```text
周期 X · 第 Y 天
part1 / part2 / part3
学习 / 复习检查
```

新增的是每个 `part` 内部的题目进度：

```text
part2 场景
题 2 / 4
```

题目内部不再出现容易重叠的“上一项 / 下一项”控制。

改为：
- 选项按钮完成题目
- 底部只出现 `检查` 或 `继续`
- 切换小题由 `继续` 驱动
- 顶部 chip 只用于内容组选择，不承担主要推进

## 10. 数据结构迁移方案

### 10.1 新增字段

在每日课程中新增：

```js
exercises: [
  {
    id: "day-1-ex-1",
    part: "foundation",
    type: "match_pairs",
    prompt: "匹配俄语和中文",
    payload: {},
    answer: {},
    feedback: {
      meaning: "中文解释",
      explanation: "简短说明"
    }
  }
]
```

### 10.2 兼容旧字段

短期保留：
- `foundation`
- `scene`
- `voice`
- `review`

新引擎优先读取 `exercises`。如果某天没有 `exercises`，继续使用旧渲染。

## 11. 实施顺序

### 阶段 1：Exercise Engine 骨架

目标：
- 支持单题渲染
- 支持进度条
- 支持检查与继续
- 支持即时反馈

范围：
- 只改第 `1` 天
- 只接入 `part1 基础`
- 题型先做 `match_pairs`
- 旧渲染作为 fallback 保留

验收：
- 第 `1` 天 `part1` 可以做匹配题
- 第 `2` 天以后仍然显示旧卡片
- `part2 / part3 / 复习检查` 不受影响
- 手机上没有重叠按钮

### 阶段 2：句子拼装题

目标：
- 支持 token bank
- 支持答案槽
- 支持正确 / 错误反馈

范围：
- 先接入第 `1` 天 `part2`
- 再扩到第 `1-3` 天

验收：
- `part2` 的句子来自 `part1` 词块
- 拼装后有即时反馈和中文含义
- 答错后允许重试

### 阶段 3：听音排序题

目标：
- 音频播放
- 慢速播放
- 点词排序

范围：
- 先接入第 `1` 天 `part2` 的第二题
- 再扩到第 `4-6` 天

### 阶段 4：复习检查题型化

目标：
- 复习不再只是“是否记住”
- 错题进入本地错题队列

范围：
- 替换普通日复习
- 第 `12` 天保留整轮复盘逻辑，但内部改题型

### 阶段 5：模板化课程扩展

目标：
- 让 `Я хочу X`、`Я не хочу X` 等模板驱动多题生成
- 建立 A1 到 A2 的课程语法路线

## 12. 对当前项目的结论

当前项目不应继续在“卡片展示 UI”上反复微调。下一步最有价值的升级是建立兼容式 `Exercise Engine`。

建议先做最小闭环：

```text
第 1 天
  part1：单词匹配
  part2：句子拼装 + 听音排序
  part3：保留当前单词 / 整句跟读
  复习检查：暂时保留确认式复习
```

只要这个闭环跑通，后续全年课程就可以逐步从“内容数据”迁移成“题型数据”。

## 13. 第一轮开发边界

第一轮只做：
- 新增 Exercise Engine 数据读取能力
- 新增 `match_pairs` 题型
- 第 `1` 天 `part1` 使用新题型
- 其他所有天保持原样

第一轮不做：
- 全年批量迁移
- 新增生命值、连击、宝箱
- 大改顶部和底部导航
- 大改第 `12` 天复盘
- 专业语音纠音
