# 多邻国式 Lesson Engine 重设计方案 v1

## 1. 目标

将当前俄语学习工具从“每日 part 卡片展示”重设计为“每日 Lesson 题流”。

新的核心体验是：

```text
第 X 轮 · 第 Y 天
  Lesson
    Exercise 1
    Exercise 2
    Exercise 3
    ...
    Exercise 8-12
  Lesson Summary
```

这更接近多邻国的学习体验：打开后直接做题，系统通过题型顺序、即时反馈和进度条控制学习节奏。

## 2. 保留与变更

### 2.1 必须保留

- 全年 `24` 个周期
- 每周期 `12` 天
- 每周期第 `12` 天为整轮复盘
- 每日 `20-30` 分钟学习时长
- 手机单题推进
- 俄语正文使用标准西里尔字母
- 俄语、发音标注、中文语义分字段保存
- 语音播放、慢速播放、跟读
- PWA 与 iPhone 主屏幕使用
- 弱游戏化，不做社交、排行榜、虚拟货币
- 不恢复手写需求

### 2.2 可以变更

- `part1 基础 / part2 场景 / part3 跟读 / 复习检查`
- 底部 `上一天 / 下一天`
- 卡片式内容展示
- 主观“是否记住”的复习页
- 当前顶部圆圈步骤

### 2.3 新核心结构

```text
Course
  └── Cycle
        └── Day
              └── Lesson
                    ├── Exercise[]
                    ├── Progress
                    ├── Instant Feedback
                    └── Summary
```

## 3. 当前实现与目标差距

当前已经具备：
- 全年课程数据
- 俄语字段规范
- 基础音频播放
- 基础跟读
- 本地进度保存
- PWA 发布

目标还缺：
- 统一题型引擎
- 每日 `8-12` 题进度条
- 即时反馈
- 句子拼装
- 听音排序
- 听写输入
- 错题回收
- 场景目标驱动的题目组合

## 4. 每日 Lesson 结构

### 4.1 普通学习日

普通日建议 `8-10` 题：

```text
1. 新词识别
2. 新词匹配
3. 听音选择
4. 句子拼装
5. 句子拼装变体
6. 听音排序
7. 中文到俄语回忆
8. 单词跟读
9. 整句跟读
10. 错题或快速复盘
```

### 4.2 第 12 天整轮复盘

第 `12` 天不学新内容，做本轮复盘：

```text
1-3. 前 11 天高频词匹配
4-6. 前 11 天核心句拼装
7-8. 听音排序
9-10. 场景复述
11-12. 本轮总结跟读
```

第 `12` 天仍然保留“本轮复盘”的概念，但内部用题流实现。

## 5. 多邻国式题型系统

### 5.1 第一批必须实现

#### 1. 单词匹配题 `match_pairs`

用途：
- 建立俄语与中文映射
- 训练快速识别

示例：

```js
{
  type: "match_pairs",
  prompt: "匹配俄语和中文",
  pairs: [
    { ru: "мама", ipa: "[MA-ma]", zh: "妈妈" },
    { ru: "тут", ipa: "[toot]", zh: "这里" }
  ]
}
```

#### 2. 句子拼装题 `sentence_builder`

用途：
- 训练语序
- 训练句型模式

示例：

```js
{
  type: "sentence_builder",
  prompt: "拼出这句话",
  answer: "Я хочу чай.",
  ipa: "[ya ha-CHOO chai]",
  zh: "我想要茶。",
  tokens: ["чай", "Я", "хочу"]
}
```

#### 3. 听音排序题 `listen_order`

用途：
- 训练听辨
- 降低完整听写难度

示例：

```js
{
  type: "listen_order",
  prompt: "按听到的顺序点词",
  answer: "Я хочу чай.",
  ipa: "[ya ha-CHOO chai]",
  zh: "我想要茶。",
  tokens: ["чай", "хочу", "Я"]
}
```

#### 4. 跟读题 `repeat_after`

用途：
- 训练模仿和语音输出

示例：

```js
{
  type: "repeat_after",
  prompt: "跟读这句话",
  answer: "Я хочу чай.",
  ipa: "[ya ha-CHOO chai]",
  zh: "我想要茶。",
  mode: "phrase"
}
```

### 5.2 第二批实现

#### 5. 听写输入题 `dictation`

用途：
- 拼写
- 主动回忆

MVP 匹配规则：
- 忽略大小写
- 忽略句号
- 可配置 `е / ё` 宽松匹配

#### 6. 中文提示输入题 `translate_to_ru`

用途：
- 从中文语义主动生成俄语

## 6. 即时反馈系统

所有题型统一使用同一反馈结构：

```text
正确 / 再试一次
正确答案
Meaning: 中文含义
简短解释
继续
```

原则：
- 答对后强化正确答案
- 答错后允许重试
- 错题进入本地错题队列
- 不用红色大面积惩罚
- 不做生命值扣减

## 7. 场景化课程设计

### 7.1 场景不是词表

多邻国式场景化应是：

```text
场景目标 → 核心词块 → 最小句型 → 句型变体 → 听说复用
```

每一天必须有明确可执行目标。

### 7.2 示例：点饮料

```text
场景目标：能表达想要某种饮料
核心词：чай / кофе / вода
核心句：Я хочу чай.
变体句：Я не хочу кофе.
听力：听 Я хочу чай 并排序
输出：跟读 Я хочу чай
复盘：中文“我想要茶”→ 俄语拼装
```

### 7.3 合格标准

- 当天词汇必须进入当天句子
- 当天句子必须进入听力和跟读
- 当天复盘必须回收词汇和句子
- 同一模板至少用 `2-3` 个词替换
- 不允许 `词汇一组、句子另一组` 的断裂设计

## 8. 语音方案

### 8.1 语音三层

#### 输入层

目标：先听懂。

题型：
- 正常播放
- 慢速播放
- 听音选择
- 听音排序

#### 模仿层

目标：能跟读。

题型：
- 单词跟读
- 整句跟读
- 慢速跟读

#### 输出层

目标：能主动说出。

题型：
- 看中文说俄语
- 看场景提示说俄语

### 8.2 MVP 规则

- 每个听力题必须有正常播放和慢速播放
- 每个跟读题必须允许“现在不方便说”
- 当前阶段继续使用浏览器 TTS / SpeechRecognition
- 不把基础识别反馈包装成专业纠音
- 后续再接入高质量音频样本

## 9. 手机界面结构

### 9.1 新页面模型

```text
顶部：
  第 X 轮 · 第 Y 天
  今日主题
  题目进度条

中间：
  当前 Exercise 卡片

底部：
  检查 / 继续
```

### 9.2 操作规则

- 一屏只出现一个核心任务
- 底部只出现一个主按钮
- `听` 按钮靠近题干，不和底部按钮抢位置
- 选项是大触控块
- 答题后才显示反馈
- 反馈后点击 `继续` 进入下一题
- 不再出现 `part1 / part2 / part3` 圆圈
- 不再出现内容项的 `上一项 / 下一项`

## 10. 数据结构

### 10.1 Day

```js
{
  day: 1,
  cycleNumber: 1,
  cycleDay: 1,
  title: "А 与 О",
  theme: "认识 А / О，并听懂最小句",
  exercises: []
}
```

### 10.2 Exercise

```js
{
  id: "c1-d1-e1",
  type: "match_pairs",
  prompt: "匹配俄语和中文",
  audio: null,
  payload: {},
  answer: {},
  feedback: {
    meaning: "中文含义",
    explanation: "简短说明"
  }
}
```

### 10.3 兼容策略

短期可以保留旧字段：
- `foundation`
- `scene`
- `voice`
- `review`

但新页面优先读取 `exercises`。

如果某天没有 `exercises`：
- 开发阶段可以自动从旧字段生成基础题
- 或临时回退旧页面

最终目标是所有天都迁移为 `exercises`。

## 11. 实施顺序

### 阶段 1：Lesson Engine 壳

目标：
- 新顶部
- Lesson 进度条
- 单题卡片
- 底部 `检查 / 继续`
- Lesson summary

范围：
- 先只实现第 `1` 天
- 其他天可暂时保持旧页面或 fallback

### 阶段 2：第一批题型

实现：
- `match_pairs`
- `sentence_builder`
- `listen_order`
- `repeat_after`

范围：
- 第 `1` 天做完整 6-8 题样板

### 阶段 3：课程数据迁移

目标：
- 从第 `1` 天扩展到第 `1` 个周期
- 第 `12` 天改为本轮题型复盘

### 阶段 4：全年迁移

目标：
- 所有 `288` 天都有 `exercises`
- 每天 `8-12` 题
- 每个周期第 `12` 天为整轮复盘题流

## 12. 第一轮开发边界

第一轮可以大改页面结构，但只作用于第 `1` 天样板。

第一轮做：
- 新增 Lesson Engine 状态
- 新增第 `1` 天 `exercises`
- 实现 `match_pairs`
- 实现基础即时反馈
- 实现进度条
- 实现结束总结页

第一轮不做：
- 全年批量迁移
- 生命值
- 宝箱
- 排行榜
- 专业语音纠音
- 复杂地图动画

## 13. 成功标准

第 `1` 天应满足：
- 打开后像一节完整小课，而不是资料页
- 用户可以连续完成题目
- 每题有明确的检查和反馈
- 俄语、发音标注、中文含义清楚
- 手机上一屏内操作不拥挤
- 不出现 `part1 / part2 / part3` 圆圈
- 不出现 `上一项 / 下一项`
