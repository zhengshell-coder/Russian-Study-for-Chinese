# 多邻国式 Lesson Engine 重设计方案 v1

## 1. 目标

将当前俄语学习工具从“每日 part 卡片展示”重设计为“每日 Lesson 题流”。

新的核心体验是：

```text
第 X 轮 · 第 Y 天
  Lesson
    Exercise 1
    Exercise 2
    Exercise 3
    ...
    Exercise 8-12
  Lesson Summary
```

这更接近多邻国的学习体验：打开后直接做题，系统通过题型顺序、即时反馈和进度条控制学习节奏。

## 2. 保留与变更

### 2.1 必须保留

- 全年 `24` 个周期
- 每周期 `12` 天
- 每周期第 `12` 天为整轮复盘
- 每日 `20-30` 分钟学习时长
- 手机单题推进
- 俄语正文使用标准西里尔字母
- 俄语、发音标注、中文语义分字段保存
- 语音播放、慢速播放、跟读
- PWA 与 iPhone 主屏幕使用
- 弱游戏化，不做社交、排行榜、虚拟货币
- 不恢复手写需求

### 2.2 可以变更

- `part1 基础 / part2 场景 / part3 跟读 / 复习检查`
- 底部 `上一天 / 下一天`
- 卡片式内容展示
- 主观“是否记住”的复习页
- 当前顶部圆圈步骤

### 2.3 新核心结构

```text
Course
  └── Cycle
        └── Day
              └── Lesson
                    ├── Exercise[]
                    ├── Progress
                    ├── Instant Feedback
                    └── Summary
```

## 3. 当前实现与目标差距

当前已经具备：
- 全年课程数据
- 俄语字段规范
- 基础音频播放
- 基础跟读
- 本地进度保存
- PWA 发布

目标还缺：
- 统一题型引擎
- 每日 `8-12` 题进度条
- 即时反馈
- 句子拼装
- 听音排序
- 听写输入
- 错题回收
- 场景目标驱动的题目组合

## 3.1 多邻国方案校准

参考多邻国公开说明后，本项目只吸收学习效率相关部分，不复制重游戏化：

- 线性路径：用户不需要每天选择学什么，系统按顺序给出下一课。
- 小课题流：打开后直接做交互题，不先阅读大段说明。
- 难度阶梯：识别 → 选择 → 排序 → 拼装 → 输入 → 跟读。
- 间隔复习：新内容和旧内容交错出现，错题优先回收。
- 场景内复用：同一课的词、句型、听力、抄写和跟读必须围绕同一目标。
- 语音从第一课开始：先听、再选、再跟读；慢速播放用于拆词和听清音节。

参考来源：
- Duolingo Method：interactive lessons、personalized practice、bite-sized lessons
- Duolingo Learning Path：线性路径、小单元、复习内置在路径里
- Duolingo Spaced Repetition：错题和旧内容按间隔回收
- Duolingo Speaking：从第一课开始做单词、句子和对话式口语练习

## 4. 每日 Lesson 结构

### 4.1 普通学习日

普通日建议 `8-12` 题：

```text
1. 新词识别
2. 俄中 / 俄英单词匹配
3. 听句子，选择听到的词
4. 听句子，按顺序点词
5. 句子拼装
6. 俄语句子抄写 / 默写
7. 中文或英文提示到俄语
8. 普通速度整句跟读
9. 慢速逐词跟读
10. 错题回收
11. 今日总结
```

### 4.2 第 12 天整轮复盘

第 `12` 天不学新内容，做本轮复盘：

```text
1-3. 前 11 天高频词匹配
4-6. 前 11 天核心句拼装
7-8. 听音排序
9-10. 场景复述
11-12. 本轮总结跟读
```

第 `12` 天仍然保留“本轮复盘”的概念，但内部用题流实现。

## 5. 多邻国式题型系统

### 5.1 第一批必须实现

#### 1. 单词匹配题 `match_pairs`

用途：
- 建立俄语与中文 / 英文映射
- 训练快速识别
- 适合食品、自我介绍、交通等场景词

示例：

```js
{
  type: "match_pairs",
  prompt: "匹配俄语和含义",
  targetLanguage: "zh",
  pairs: [
    { ru: "мама", ipa: "[MA-ma]", zh: "妈妈", en: "mom" },
    { ru: "тут", ipa: "[toot]", zh: "这里", en: "here" }
  ]
}
```

说明：
- 中文是主语义锚点
- 英文作为辅助桥梁，适合用户已掌握英语的情况
- 同一数据项同时保留 `zh` 和 `en`，题型可切换俄中或俄英

#### 2. 听句选词题 `listen_choose_words`

用途：
- 听一个完整句子
- 从候选词中选出听到的词或词组
- 比直接听写更适合初学者

示例：

```js
{
  type: "listen_choose_words",
  prompt: "听句子，选择你听到的词",
  audioText: "Я хочу чай.",
  ipa: "[ya ha-CHOO chai]",
  zh: "我想要茶。",
  correctTokens: ["Я", "хочу", "чай"],
  choices: ["чай", "кофе", "Я", "не", "хочу"]
}
```

#### 3. 听音排序题 `listen_order`

用途：
- 训练听辨
- 训练词序
- 降低完整听写难度

示例：

```js
{
  type: "listen_order",
  prompt: "按听到的顺序点词",
  answer: "Я хочу чай.",
  ipa: "[ya ha-CHOO chai]",
  zh: "我想要茶。",
  tokens: ["чай", "хочу", "Я"]
}
```

#### 4. 句子拼装题 `sentence_builder`

用途：
- 训练语序
- 训练句型模式

示例：

```js
{
  type: "sentence_builder",
  prompt: "拼出这句话",
  answer: "Я хочу чай.",
  ipa: "[ya ha-CHOO chai]",
  zh: "我想要茶。",
  tokens: ["чай", "Я", "хочу"]
}
```

#### 5. 俄语抄写 / 默写题 `copy_or_type_ru`

用途：
- 看俄语句子，手动输入一遍
- 或听俄语后输入一遍
- 强化字母、拼写和词形记忆

示例：

```js
{
  type: "copy_or_type_ru",
  prompt: "把这句俄语输入一遍",
  answer: "Я хочу чай.",
  ipa: "[ya ha-CHOO chai]",
  zh: "我想要茶。",
  showAnswerBeforeTyping: true
}
```

#### 6. 跟读题 `repeat_after`

用途：
- 训练模仿和语音输出
- 区分普通连读和慢速逐词

示例：

```js
{
  type: "repeat_after",
  prompt: "跟读这句话",
  answer: "Я хочу чай.",
  ipa: "[ya ha-CHOO chai]",
  zh: "我想要茶。",
  mode: "phrase",
  slowMode: "word_by_word"
}
```

规则：
- 普通播放：整句连起来读
- 慢速播放：按词播放，词与词之间有停顿
- 跟读可分为单词跟读和整句跟读

### 5.2 第二批实现

#### 7. 完整听写输入题 `dictation`

用途：
- 拼写
- 主动回忆

MVP 匹配规则：
- 忽略大小写
- 忽略句号
- 可配置 `е / ё` 宽松匹配

#### 8. 中文 / 英文提示输入题 `translate_to_ru`

用途：
- 从中文或英文语义主动生成俄语

说明：
- 中文用于确认意思
- 英文可作为辅助桥梁
- 不能让英文替代俄语学习目标

## 6. 即时反馈系统

所有题型统一使用同一反馈结构：

```text
正确 / 再试一次
正确答案
Meaning: 中文含义
简短解释
继续
```

原则：
- 答对后强化正确答案
- 答错后允许重试
- 错题进入本地错题队列
- 不用红色大面积惩罚
- 不做生命值扣减

## 7. 场景化课程设计

### 7.1 场景不是词表

多邻国式场景化应是：

```text
场景目标 → 核心词块 → 最小句型 → 句型变体 → 听说复用
```

每一天必须有明确可执行目标。

### 7.2 场景目标不做独立说明页

每个 Lesson 必须有清晰场景目标，但不单独做“今日场景目标”说明页。

正确做法：
- 顶部只保留短主题，如“点饮料”
- 第一题直接进入词汇识别或听句选词
- 后续题目反复使用同一批词和同一句型
- 场景目标通过题目顺序自然体现，不把设计思路展示给用户

这样既保留课程主线，又避免页面变成说明文档。

### 7.3 示例：点饮料

```text
场景目标：能表达想要某种饮料
核心词：чай / кофе / вода
核心句：Я хочу чай.
变体句：Я не хочу кофе.
听力 1：听 Я хочу чай 并选择听到的词
听力 2：听 Я хочу чай 并排序
拼写：抄写 Я хочу чай.
输出：普通速度跟读 + 慢速逐词跟读
复盘：中文 / 英文“我想要茶 / I want tea”→ 俄语拼装
```

### 7.4 合格标准

- 当天词汇必须进入当天句子
- 当天句子必须进入听力和跟读
- 当天复盘必须回收词汇和句子
- 同一模板至少用 `2-3` 个词替换
- 不允许 `词汇一组、句子另一组` 的断裂设计
- 每天必须有明确主题，如食品、自我介绍、时间、问路

## 8. 语音方案

### 8.1 语音三层

#### 输入层

目标：先听懂。

题型：
- 正常播放
- 慢速播放
- 听音选择
- 听音排序

#### 模仿层

目标：能跟读。

题型：
- 单词跟读
- 整句跟读
- 慢速跟读

#### 输出层

目标：能主动说出。

题型：
- 看中文说俄语
- 看场景提示说俄语

### 8.2 MVP 规则

- 每个听力题必须有正常播放和慢速播放
- 普通播放是完整连读
- 慢速播放应尽量逐词播放
- 每个跟读题必须允许“现在不方便说”
- 当前阶段继续使用浏览器 TTS / SpeechRecognition
- 不把基础识别反馈包装成专业纠音
- 后续再接入高质量音频样本

### 8.3 慢速逐词播放规则

对句子：

```text
Я хочу чай.
```

普通播放：

```text
Я хочу чай.
```

慢速播放：

```text
Я    хочу    чай
```

实现方式：
- 将句子按空格拆成词
- 每个词单独播放
- 词之间增加短暂停顿
- 如果浏览器不稳定，则先用较低 rate 播放整句作为 fallback

## 9. 手机界面结构

### 9.1 新页面模型

```text
顶部：
  第 X 轮 · 第 Y 天
  今日主题
  题目进度条

中间：
  当前 Exercise 卡片

底部：
  检查 / 继续
```

### 9.2 操作规则

- 一屏只出现一个核心任务
- 底部只出现一个主按钮
- `听` 按钮靠近题干，不和底部按钮抢位置
- 选项是大触控块
- 答题后才显示反馈
- 反馈后点击 `继续` 进入下一题
- 不再出现 `part1 / part2 / part3` 圆圈
- 不再出现内容项的 `上一项 / 下一项`

## 10. 数据结构

### 10.1 Day

```js
{
  day: 1,
  cycleNumber: 1,
  cycleDay: 1,
  title: "А 与 О",
  theme: "认识 А / О，并听懂最小句",
  exercises: []
}
```

### 10.2 Exercise

```js
{
  id: "c1-d1-e1",
  type: "match_pairs",
  prompt: "匹配俄语和中文",
  audio: null,
  payload: {},
  answer: {},
  feedback: {
    meaning: "中文含义",
    explanation: "简短说明"
  }
}
```

### 10.3 兼容策略

短期可以保留旧字段：
- `foundation`
- `scene`
- `voice`
- `review`

但新页面优先读取 `exercises`。

如果某天没有 `exercises`：
- 开发阶段可以自动从旧字段生成基础题
- 或临时回退旧页面

当前实现已为所有 `288` 天生成 `exercises`。旧字段继续保留为课程数据来源和兼容层，但页面优先读取题流。

范围：
- 第 `1` 天
- 加入 `listen_choose_words`
- 加入 `copy_or_type_ru`

### 阶段 2：第一批题型稳定

范围：
- 第 `1` 天做完整 `8-10` 题样板

### 阶段 3：课程数据迁移

目标：
- 已完成全年自动题流生成
- 第 `12` 天已改为本轮题型复盘

目标：
- 所有 `288` 天都有 `exercises`
- 每天 `8` 题

## 12. 第一轮开发边界

第一轮可以大改页面结构，但只作用于第 `1` 天样板。

第一轮做：
- 新增 Lesson Engine 状态
- 新增第 `1` 天 `exercises`
- 实现 `match_pairs`
- 实现基础即时反馈
- 实现进度条
- 实现结束总结页

下一轮优先补：
- `listen_choose_words`
- `copy_or_type_ru`
- 慢速逐词播放
- 俄中 / 俄英双语匹配模式

第一轮不做：
- 全年批量迁移
- 生命值
- 宝箱
- 排行榜
- 专业语音纠音
- 复杂地图动画

## 13. 成功标准

第 `1` 天应满足：
- 打开后像一节完整小课，而不是资料页
- 用户可以连续完成题目
- 每题有明确的检查和反馈
- 俄语、发音标注、中文含义清楚
- 手机上一屏内操作不拥挤
- 不出现 `part1 / part2 / part3` 圆圈
- 不出现 `上一项 / 下一项`
