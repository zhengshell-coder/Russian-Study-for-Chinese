# 更改记录 2026-04-18 · 俄语内容规范修正

## 1. 本次修正目的
本次修正用于解决两类问题：

1. 俄语内容展示不够严谨  
表现为把俄文字母和拉丁字母混用，或在说明字段中混入不规范写法。

2. 场景交互行为不一致  
表现为用户点击“听”时，页面发生了跳转，而不是只播放音频。

## 2. 已完成修正

### 2.1 俄语文本规则
- 所有俄语正文统一使用标准俄语西里尔字母
- 拉丁转写统一放在 `translit`
- 中文说明统一放在 `zh`、`nameZh`、`note`
- 不再允许把俄文字母当成拉丁字母替代展示
- 明确加入 `Л л`，并说明它对应转写 `l`，不是拉丁 `L`
- 字母课程文档中的“名称”统一改为 `标准名称（俄语）`
- 若需要中文辅助，只能单独放在 `中文提示`

### 2.2 场景交互规则
- 场景模块中所有按钮统一使用 `听`
- 点击场景中的词或句子时，只执行播放
- 不再因为点击“听”而自动跳转到跟读页
- 跟读页只由学习路径进入，不由场景播放动作强制跳转

### 2.3 显示稳定性修正
- 为俄语正文增加单独的 `.ru-text` 字体规则
- 目的是尽量减少系统字体对西里尔字母外观的干扰

## 3. 防止同类错误的检查清单

后续每次改动俄语内容时，必须逐项检查：

1. `word` / `phrase.ru` / `example.word` 是否全部为标准俄文
2. `translit` 是否只含拉丁转写
3. `nameZh` / `zh` / `note` 是否只含中文说明
4. 是否把“相似字形”误当成“同一个字母”
5. 场景里的“听”按钮是否只播放，不跳转
6. 字母卡中是否出现不必要的混合说明
7. 33 个字母的标准俄语名称是否逐个核对过

## 4. 当前明确执行规则

### 4.1 数据字段规则
- `uppercase` / `lowercase`：只存俄文字母
- `example.word`：只存俄文单词
- `phrase.ru`：只存俄文句子
- `translit`：只存拉丁转写
- `nameZh` / `zh` / `note`：只存中文说明
- 字母课程文档中的 `标准名称（俄语）`：只存标准俄语字母名称
- 字母课程文档中的 `中文提示`：只存中文辅助说明

### 4.2 页面行为规则
- `听`：只播放
- `开始跟读`：只在跟读模块内触发
- 不用文案“用这句”这种带行为歧义的按钮

## 5. 后续执行方式
- 后续每轮如果修改字母数据，必须在提交前抽检至少一遍全部字母
- 后续每轮如果修改场景模块，必须手测“听”按钮行为
- 若出现俄语显示异常，先确认字符本身，再确认字体渲染，不允许先猜内容错误
